# 시작하기 — HSM 연결
모든 기능은 HSM(슬롯) 로그인 후에 동작합니다.

[따라하기]
1) [연결] 탭으로 이동합니다.
2) '슬롯'에 파티션 슬롯 번호를 입력합니다 (예: 0).
3) 'PIN'에 파티션 비밀번호(crypto user PIN)를 입력합니다.
4) [연결] 버튼을 누릅니다.
5) 상단 우측 표시가 초록색 "● 연결됨: <토큰라벨>"로 바뀌면 성공입니다.
6) 아래 '키 목록'에 토큰의 키가 표시됩니다.

[확인/문제 해결]
- "PIN이 올바르지 않습니다" → PIN 재확인. 여러 번 틀리면 슬롯이 잠길 수 있습니다.
- 연결이 안 되면 LunaClient 설치 경로/슬롯 번호를 확인하세요.
- 작업이 끝나면 [연결 해제]로 로그아웃합니다.

※ PIN은 저장되지 않습니다. 매 실행 시 다시 입력합니다.


# 메커니즘 탐색기 — 무엇을 쓸 수 있는지 먼저 확인
HSM 펌웨어/Provider가 실제로 지원하는 알고리즘을 한눈에 봅니다.

[따라하기]
1) [메커니즘 탐색기] 탭으로 이동합니다.
2) 연결되어 있으면 '가용' 열에 ✅(사용 가능) / 🔒(현재 미지원)이 표시됩니다.
3) 상단 검색창에 이름 일부를 입력해 거를 수 있습니다 (예: "AES", "RSA", "SHA").
4) 'Family'/'Op' 콤보로 종류·연산별로 필터링합니다.
5) '가용한 것만' 체크 시 현재 HSM이 지원하는 항목만 보입니다.

[플래그 의미]
- 🛡 FIPS 승인 알고리즘
- 🏷 벤더(Thales/SafeNet) 확장
- ⚠ legacy/취약 — 신규 사용 비권장
- 🌏 지역/도메인 한정 (ARIA=한국, SM=중국, DUKPT=결제 등)

※ 연산 탭(암복호화/서명/MAC)에는 표준(Tier A) 알고리즘이 항상 보이고,
  PQC·SM 등은 HSM이 실제로 노출할 때만 나타납니다.


# 키 생성 — AES / RSA / EC / DES3 / ARIA
연산에 쓸 키를 토큰에 만듭니다.

[따라하기]
1) [키 관리] 탭 상단 '키 생성' 영역으로 갑니다.
2) '알고리즘'을 선택합니다 (AES, RSA, EC, DES3, ARIA).
3) '키 길이/곡선'을 선택합니다.
   - AES: 128/192/256
   - RSA: 2048/3072/4096
   - EC: secp256r1/secp384r1/secp521r1
   - DES3: 168 (legacy)
   - ARIA: 128/192/256
4) '라벨(alias)'에 키 이름을 입력합니다 (예: demo-aes, demo-rsa).
5) [키 생성]을 누릅니다.
6) 아래 '토큰 키 목록'에 새 키가 나타납니다.

[중요 — RSA/EC 키쌍의 라벨 규칙]
- 비대칭 키는 개인키=<라벨>, 공개키=<라벨>-pub 로 분리 저장됩니다.
  예) 라벨 "demo-rsa" → 개인키 "demo-rsa", 공개키 "demo-rsa-pub"
- 암복호화/서명 시 둘 중 무엇을 골라도 프로그램이 짝을 자동으로 찾습니다.

※ 새로 만든 키는 다른 탭(암복호화/MAC/서명) 키 목록에 자동 반영됩니다.


# AES 암복호화 (CBC / GCM / CTR 등)
대칭키로 평문을 암호화하고 다시 복호화합니다.

[따라하기 — 암호화]
1) [암복호화] 탭으로 갑니다.
2) '메커니즘'에서 AES 모드를 선택합니다.
   - AES_GCM   : 인증 암호(AEAD), 권장
   - AES_CBC_PKCS5 : 일반 블록암호
   - AES_CTR   : 스트림 방식
3) '키'에서 AES 키를 선택합니다.
4) 왼쪽 '평문' 칸에 텍스트를 입력합니다.
5) [암호화]를 누르면 가운데 '암호문(Base64)' 칸에 결과가 나옵니다.

[따라하기 — 복호화]
6) '암호문' 칸의 Base64 값을 그대로 두고(또는 붙여넣고) [복호화]를 누릅니다.
7) 오른쪽 '복호문' 칸에 원문이 복원됩니다.

[한 번에 검증]
- [라운드트립 검증]은 암호화→복호화를 자동 수행하고 ✓PASS/✗FAIL을 표시합니다.

※ IV(초기화벡터)는 자동 생성되어 암호문 앞에 [IV][암호문] 형태로 붙습니다.
  외부 AES 암호문을 복호화하려면 이 형식이어야 합니다.


# RSA 암복호화 — 공개키/개인키 방향
공개키로 암호화하고 개인키로 복호화하는 것이 기본입니다.

[따라하기 — 기본(공개키 암호화)]
1) [암복호화] 탭에서 메커니즘 'RSA_OAEP_SHA256'(권장) 또는 'RSA_PKCS_ENC'를 선택합니다.
2) '키'에서 RSA 키(키쌍 또는 -pub)를 선택합니다.
3) 'RSA 키 방향' 체크박스는 해제 상태로 둡니다.
4) '평문'에 텍스트 입력 후 [암호화] → '암호문'에 Base64 결과.
5) 복호화하려면 [복호화] → '복호문'에 원문 복원.
   (암호화=공개키, 복호화=개인키를 자동으로 사용)

[개인키로 암호화 ↔ 공개키로 복호화]
- 'RSA 키 방향' 체크박스를 켜면 방향이 반대가 됩니다(암호화=개인키, 복호화=공개키).
- 이때 메커니즘은 반드시 'RSA_PKCS_ENC'를 쓰세요.
  (OAEP는 공개키 암호화 전용이라 개인키 암호화에서 실패합니다.)

[암호문만 받아서 외부에서 쓰기]
- 평문 + 공개키 → [암호화] → 나온 Base64 암호문을 복사해 사용합니다.
- OAEP는 매번 랜덤 패딩이라 같은 평문도 암호문이 달라집니다(정상).

[암호문만 복호화 테스트]
- '암호문' 칸에 Base64 암호문을 직접 붙여넣고, 키 선택 후 [복호화]만 눌러도 됩니다.
  (원문 입력 없이 복호화만 가능)

※ RSA는 평문 길이에 제한이 있습니다(키 크기 - 패딩). 긴 데이터는
  AES로 암호화하고 그 AES 키만 RSA로 감싸는 하이브리드를 권장합니다.


# 서명 · 검증 (성능 측정 포함)
개인키로 서명하고 공개키로 검증하며, 처리량을 측정합니다.

[따라하기]
1) [서명·검증] 탭으로 갑니다.
2) '메커니즘'에서 서명 알고리즘을 선택합니다.
   - SHA256withRSA, SHA256withRSA/PSS, SHA256withECDSA, Ed25519 등
3) '키'에서 해당 키(개인키/키쌍)를 선택합니다.
4) 속도(초당 횟수) 슬라이더를 조절합니다.
5) '외부 검증' 체크 시 표준 Provider(SunRsaSign 등)로 독립 검증합니다.
6) [시작]을 누르면 루프가 돌며 통계(총/성공/실패/평균 시간)가 갱신됩니다.
7) [중지] 후 [JSON 저장]/[CSV 저장]으로 결과를 보관할 수 있습니다.

※ 외부 검증은 HSM 결과를 JVM 표준 Provider로 다시 확인하는 교차검증입니다.


# MAC · 해시
메시지 인증코드(MAC)와 해시값을 계산합니다.

[따라하기]
1) [MAC·해시] 탭으로 갑니다.
2) '메커니즘'을 선택합니다.
   - MAC(키 필요): HmacSHA256, AESCMAC, AESGMAC 등
   - 해시(키 불필요): SHA-256, SHA-512, SHA3-256 등
3) MAC을 고른 경우 '키(MAC용)'에서 대칭키를 선택합니다.
   (해시를 고르면 키 선택은 비활성화됩니다.)
4) 왼쪽 '입력' 칸에 텍스트를 입력합니다.
5) [계산]을 누르면 오른쪽에 16진수(HEX) 결과와 바이트 수·소요시간이 표시됩니다.

※ 해시는 키 없이 누구나 같은 값을 얻습니다. MAC은 키를 아는 측만 검증 가능합니다.


# 키 래핑 / 언래핑 (AES-KWP)
키를 다른 키(AES)로 감싸 안전하게 내보내고, 다시 가져옵니다.

[따라하기 — 래핑/내보내기]
1) [키 관리] 탭 '키 래핑/내보내기(AES-KWP)' 영역으로 갑니다.
2) '래핑 키(AES)'에서 감쌀 때 쓸 AES 키를 선택합니다.
3) 위 '토큰 키 목록'에서 래핑 대상 키를 클릭해 선택합니다.
4) [래핑 후 내보내기]를 누르면 .key 파일로 저장됩니다.

[따라하기 — 언래핑/가져오기]
5) '키 언래핑/가져오기(AES-KWP)' 영역에서 동일한 AES 키를 선택합니다.
6) [파일 선택]으로 .key 파일을 고릅니다.
7) 복원할 키의 새 라벨을 입력하고 [언래핑/가져오기]를 누릅니다.

※ 개인키를 래핑하려면 생성 시 CKA_EXTRACTABLE과 파티션 정책이 허용되어야 합니다.


# 공개키 내보내기
토큰의 공개키를 표준 파일(PEM/DER)로 추출합니다.

[따라하기]
1) [키 관리] 탭 '토큰 키 목록'에서 비대칭 키(키쌍/공개키/개인키)를 선택합니다.
2) 상단의 [공개키 내보내기] 버튼을 누릅니다.
3) 저장 형식을 고릅니다.
   - .pem : 텍스트(-----BEGIN PUBLIC KEY-----), 가장 호환성 높음
   - .der : 바이너리(X.509 SubjectPublicKeyInfo)
4) 저장하면 다른 시스템에서 그대로 사용할 수 있습니다.

※ 어느 라벨(개인키/공개키/키쌍)을 골라도 짝 공개키를 자동으로 찾아 내보냅니다.
  공개키는 비밀이 아니므로 래핑 없이 추출됩니다.


# 키 속성 · 라벨 변경 · 삭제
토큰 키의 PKCS#11 속성을 보고 바꾸거나, 라벨 변경·삭제합니다.

[속성 보기/변경]
1) [키 관리] 탭 '토큰 키 목록'에서 키를 클릭합니다.
2) '속성(PKCS#11)' 영역에 체크박스로 현재 속성이 표시됩니다.
   (SIGN/DECRYPT/UNWRAP/EXTRACTABLE 등 — 키에 없는 속성은 비활성)
3) 원하는 값으로 체크/해제 후 [속성 적용]을 누릅니다.

[라벨 변경 / 삭제]
- [라벨 변경]: 새 라벨 입력 후 변경. 키쌍은 -pub 짝도 함께 관리하세요.
- [삭제]: 선택 키를 토큰에서 영구 삭제합니다(되돌릴 수 없음, 확인창 표시).

※ CKA_EXTRACTABLE은 PKCS#11 규칙상 false→true 변경이 불가합니다(생성 시 결정).


# 양자내성암호 (PQC) — ML-DSA / ML-KEM
실제 HSM 토큰에 PQC 키쌍을 만들고 그 키로 서명/캡슐화합니다. (HSM 연결 필요)

[따라하기 — 키 한 번 생성 (재사용)]
1) [연결] 탭에서 HSM에 연결합니다.
2) [PQC] 탭 위쪽 '알고리즘'에서 선택합니다.
   - [서명] ML-DSA-44/65/87  → 서명/검증
   - [KEM]  ML-KEM-512/768/1024 → 키 캡슐화
3) '라벨(alias)'에 키 이름을 입력하고 [토큰에 키 생성]을 누릅니다.
   (개인키=<라벨>, 공개키=<라벨>-pub 로 토큰에 영구 저장)

[따라하기 — 실행 (기존 키 재사용)]
4) 아래 '토큰 키'에서 사용할 PQC 키를 선택합니다.
5) (ML-DSA만) 왼쪽 '서명할 메시지'에 텍스트를 입력합니다.
6) [실행]을 누릅니다.
   - ML-DSA: 서명 길이·시간 + 검증 ✓PASS
   - ML-KEM: 캡슐화→디캡슐화 후 공유비밀 일치 ✓PASS (HMAC 지문으로 비교)
- 같은 키로 [실행]을 반복할 수 있습니다. 키가 안 보이면 [키 새로고침].

※ 키 관리 탭에서도 알고리즘 ML_DSA/ML_KEM 으로 PQC 키를 만들 수 있습니다.
※ ML-DSA=서명(RSA/ECDSA 대체), ML-KEM=키 교환(ECDH 대체).
  표준: ML-DSA(FIPS 204), ML-KEM(FIPS 203). 서명 연산명은 "ML-DSA"입니다.
※ 공유비밀은 토큰 밖으로 추출되지 않으므로 HMAC-SHA256 지문으로 일치를 확인합니다.


# 자주 묻는 질문 / 팁
[Q] 연산 탭에 원하는 알고리즘이 안 보여요.
 - 표준 알고리즘(RSA/AES/EC 등)은 항상 보입니다. PQC·SM 등 특수 항목은
   HSM이 실제 노출할 때만 나타납니다. [메커니즘 탐색기]에서 가용성을 확인하세요.

[Q] 같은 평문인데 암호문이 매번 달라요.
 - 정상입니다. GCM/CBC는 랜덤 IV, RSA-OAEP는 랜덤 패딩을 씁니다.

[Q] 키를 만들었는데 다른 탭에 안 보여요.
 - 자동 반영됩니다. 그래도 없으면 해당 탭에서 키를 다시 선택하거나
   [키 관리] 탭의 [새로고침]을 눌러보세요.

[Q] RSA로 긴 텍스트가 암호화가 안 돼요.
 - RSA는 키 크기에서 패딩을 뺀 길이까지만 가능합니다. 긴 데이터는
   AES로 암호화하고 AES 키만 RSA로 감싸는 하이브리드를 쓰세요.
